Сортировки:
Пирамидальная (heap sort)
Сортировка выбором (selection sort)
Быстрая (quick sort)

Массив данных по цветам: название цветка, цвет, аромат (сильный, умеренный, слабый), регионы распространения (сравнение по полям - название, цвет, аромат)

Данные для сортировки генерируются по ходу выполнения программы. Время работы каждой сортировки для данных каждого объема записывается в файл data.csv.

Далее с помощью MS Excel был сгенерирован график (ось y отображается в логарифмическом масштабе из-за большого диапазона значений):

Согласно теоретическим данным, сортировки имеют следующую асимптотическую сложность:

  • Сортировка выбором: в худшем, среднем и лучшем случае O(n^2).
  • Пирамидальная: в худшем, среднем и лучшем случае O(nlogn).
  • Быстрая: в худшем случае может достигать O(n^2) (маловероятно), в лучшем и среднем случае O(nlogn).
На графике видно, что быстрая и пирамидальная практически совпадают по времени работы, а сортировка выбором работает намного медленнее. Это совпадает с теоретическими данными.
